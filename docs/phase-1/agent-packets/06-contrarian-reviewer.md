# Packet 06 — Contrarian Reviewer

## Input context

- Product thesis
- Phase -1 artifacts
- Pass/adapt/stop criteria
- Any completed session evidence

## Exact task

Make the strongest evidence-based case that this wedge should not be built, then state what evidence would change that conclusion.

## In scope

- Existing-tool sufficiency
- Workflow frequency
- Review burden
- Confidentiality and integration barriers
- Legal/compliance attractiveness
- Alternative wedge: clause pack, covenant calendar, or human-led service

## Out of scope

- Cynicism without evidence
- Invented competitor claims
- Product implementation

## Owned files

- Future output: `docs/phase-1/contrarian_review.md`

## Expected output and acceptance criteria

- At least five distinct no-build arguments
- Evidence status for each: observed, inferred, or untested
- Strongest alternative explanation for positive feedback
- Specific evidence that would falsify each major objection
- Clear recommendation: proceed, adapt, stop, or insufficient evidence

## Completion evidence

- Evidence table
- Unresolved assumption list
- Description of the cheapest disconfirming test

## Stop and escalate when

- Available evidence is too thin to support a recommendation
- The requested conclusion is predetermined
- External competitor facts would require unverified claims

## Do not

- Treat enthusiasm as adoption
- Treat all human review as failure; test whether it is the correct product design
- recommend more building merely to gather demand evidence
